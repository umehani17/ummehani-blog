Building My First Lab Website

By Umme Hani • 20 Mar 2026 • 6 min read

This was my first full website for university lab submission. I used only HTML, CSS, and small JavaScript. My goal was to keep the page simple, readable, and responsive.

Project structure I used

I avoided complex setup and created a small folder with clear file names. This helped me stay organized while learning.

project-files.txt 
# Beginner website structure
index.html
style.css
script.js
post1.html
post2.html
post3.html

How I started the layout

I first built the basic HTML sections: header, hero, post cards, and footer. After this, I added CSS to improve colors and spacing.

index.html (basic layout) 
<header>...</header>
<main>
  <section>Hero content</section>
  <section>Blog cards</section>
</main>
<footer>Footer links</footer>

What I learned

  • Simple structure is easier to debug.
  • Good spacing makes a design look cleaner.
  • Mobile testing is important even for small projects.
  • Small JavaScript features can improve user experience.

Problems I faced and how I solved them

At first, my sections were not aligned properly and card spacing looked uneven. I solved this by using one container width and consistent padding values in all sections.

I also checked the website on phone view and found text was too large in some places. After adding responsive CSS media queries, the page looked much better on small screens.

Final result of this lab task

This project helped me understand how to create a full multi-page website without using frameworks. I feel more confident now to build the next project with better structure and cleaner styling.
